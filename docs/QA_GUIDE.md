# 🧪 SoundLab QA 가이드 (Quality Assurance Guide)

연구원이 아니더라도 **SoundLab**이 정상적으로 작동하는지 확인할 수 있는 테스트 시나리오입니다. 아래 순서대로 앱을 테스트해보세요.

---

## 1. 기본 실행 및 업로드 (Basic Functionality)
가장 기초적인 기능이 작동하는지 확인합니다.

### ✅ 체크리스트
- [ ] **앱 실행**: `streamlit run src/main.py` 명령어로 앱이 브라우저에서 열리는가? (오류 메시지 없음)
- [ ] **파일 업로드**:
    - `.wav` 파일을 드래그 앤 드롭했을 때 "신호 처리 중..." 스피너가 돌다가 사라지는가?
    - **통과 기준**: 에러(빨간 박스) 없이 차트가 화면에 나타나야 함.

---

## 2. V5.7 스마트 분석 검증 (Smart Analysis Logic)
새로 적용된 **V5.7 알고리즘**이 제대로 돌아가는지 "현상" 위주로 확인합니다.

### ✅ 체크리스트
- [ ] **스마트 모드 토글**:
    - 사이드바의 `🧠 스마트 분석 모드`가 기본적으로 **켜져(ON)** 있는가?
    - 토글을 **껐을 때(OFF)**: 차트의 형태가 변하는가? (스마트 모드는 조용한 구간을 잘라내므로, 끄면 차트가 더 길게 보여야 함)
- [ ] **자동 대역폭 감지**:
    - 파일 업로드 후 `📊 스마트 분석 결과` 섹션에 `감지된 대역폭 (Bandwidth)` 수치가 뜨는가? (예: `10.00 Hz` 등)
- [x] **안전 절삭(Safe Trimming)**:
    - 차트의 시작 부분이 0초가 아닌 경우(예: 60초 이후부터 시작)가 있는가?
        - **설명**: V5.7은 기계가 켜진 후 첫 1분(Safety Buffer)을 무시하거나, 소음 구간을 자동으로 잘라냅니다. 데이터가 있다면 정상입니다.
- [ ] **타임라인 (Timeline)**:
    - **가동 타임라인 (Timeline)**이라는 가로 막대 차트가 보이는가?
    - `총 가동 시간`과 `가동 횟수`가 숫자로 표시되는가?
    - 아래의 `상세 가동 이력` 표에 시작/종료 시간이 분:초 단위로 잘 나오는가?


---

## 3. UI 가독성 및 반응성 (UI/UX)
사용자가 보기에 편한지 확인합니다.

### ✅ 체크리스트
- [ ] **시간 표시 (MM:SS)**:
    - 로그 테이블과 차트 위를 마우스로 가리켰을 때, 시간이 `300.0` 같은 숫자 대신 **`05:00` (분:초)** 형식으로 나오는가?
- [ ] **컬럼명 확인**:
    - 테이블 헤더가 `시간(초)`가 아닌 **`발생 시각 (MM:SS)`** 등으로 친절하게 바뀌었는가?
- [ ] **인터랙션 연동**:
    - 테이블의 행(Row)을 클릭했을 때, 위쪽 차트에 **빨간색 세로선**이 생기는가?
    - `Shift` 키를 누르고 여러 행을 동시에 선택할 수 있는가?

---

## 4. 스트레스 테스트 (Stability)
앱이 튼튼한지 확인합니다.

### ✅ 체크리스트
- [ ] **슬라이더 마구 움직이기**:
    - `민감도 (Otsu 계수)` 슬라이더를 빠르게 좌우로 움직였을 때 앱이 멈추지 않고 차트의 붉은 선(임계값)이 바로바로 따라오는가?
- [ ] **새로고침**:
    - 브라우저를 새로고침(F5) 했을 때, 이전에 설정한 값들이 초기화되더라도 앱이 죽지 않고 다시 로드되는가?

## 📝 QA 결과 리포트 양식 (예시)

| 항목 | 결과 (Pass/Fail) | 비고 |
| :--- | :---: | :--- |
| **기본 실행** | Pass | 정상 구동됨 |
| **V5.7 로직** | Pass | 스마트 모드 꼈을 때 차트 변화 확인됨 |
| **가독성** | Pass | 05:30 형식으로 잘 보임 |
| **안정성** | Fail | 슬라이더를 너무 빨리 움직이니 잠깐 멈춤 |

> **Tip**: "Fail"이 나오면 해당 상황을 캡처해서 개발자에게 전달해주세요! 📸
